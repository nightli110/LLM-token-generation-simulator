## 数据概述

项目包含两大类硬件的基准测试数据：

1. **Apple Silicon** - M1、M2、M3 系列芯片
2. **NVIDIA GPU** - RTX 30/40 系列、专业卡、数据中心 GPU

每个设备都有两种类型的测试数据：
- **Processing** - 处理速度（tokens/s）
- **Generation** - 生成速度（tokens/s）

---

## 性能指标解释

### Processing (处理速度)

指模型在处理提示（prompt）时的吞吐量，通常也称为"预填充"（prefill）阶段。

**特点**:
- 通常比生成速度快很多
- 可以并行处理整个输入序列
- 受内存带宽影响极大
- 数值通常在 1000-10000+ tokens/s

**用途**:
- 衡量模型处理长上下文的能力
- 评估批量处理性能

### Generation (生成速度)

指模型逐个生成输出令牌的速度，这是用户实际感知到的"打字"速度。

**特点**:
- 通常比处理速度慢很多
- 顺序生成，难以并行化
- 更能代表实际用户体验
- 数值通常在 10-200 tokens/s

**用途**:
- 衡量实时交互体验
- 评估对话式 AI 的响应速度

## 模型和量化

### 支持的模型

| 模型 | 参数量 | 说明 |
|------|--------|------|
| Llama2 | 7B | Meta 的开源模型 |
| Llama3 | 8B | Meta 的最新模型 |
| Llama3 | 70B | 大型模型，性能更强 |

### 量化级别

| 类型 | 说明 | 显存需求 | 质量损失 |
|------|------|----------|----------|
| F16 | 16位浮点数（全精度） | 高 | 无 |
| Q8_0 | 8位量化 | 中 | 轻微 |
| Q4_0 | 4位量化 | 低 | 中等 |
| Q4_K_M | 4位 K-quant 量化 | 低 | 轻微 |

**量化的权衡**:
- 更低的量化 = 更快的速度 + 更少的显存使用 + 更低的质量
- 更高的量化 = 更慢的速度 + 更多的显存使用 + 更好的质量

### 命名约定

模型数据字段命名格式：

```
{模型}_{参数量}_{量化类型}_{测试类型}
```

示例：
- `Llama2_7B_Q4_0_Generation` - Llama2 7B 模型，Q4_0 量化，生成速度
- `Llama3_70B_Q4_K_M_Processing` - Llama3 70B 模型，Q4_K_M 量化，处理速度

## 设备规格

### Apple Silicon

Apple 设备包含两个关键规格：

| 指标 | 说明 | 影响 |
|------|------|------|
| GPUCores | GPU 核心数 | 更多核心通常 = 更快速度 |
| BW (Bandwidth) | 内存带宽 (GB/s) | LLM 推理受内存带宽限制极大 |

**Apple Silicon 内存带宽参考**:

| 芯片 | 内存带宽 |
|------|----------|
| M1 | 68 GB/s |
| M1 Pro | 200 GB/s |
| M1 Max | 400 GB/s |
| M1 Ultra | 800 GB/s |
| M2 | 100 GB/s |
| M2 Pro | 200 GB/s |
| M2 Max | 400 GB/s |
| M2 Ultra | 800 GB/s |
| M3 | 100 GB/s |
| M3 Pro | 150 GB/s |
| M3 Max | 400 GB/s |

### NVIDIA GPU

NVIDIA 设备以显存容量标识（如 `3070_8GB`、`4090_24GB`）。

**关键 NVIDIA GPU 系列**:

- **RTX 30 系列** - Ampere 架构
- **RTX 40 系列** - Ada Lovelace 架构
- **RTX 专业卡** - 工作站显卡
- **A 系列** - 数据中心 GPU (A10, A40, A100)
- **L 系列** - 数据中心 GPU (L40S)

## 数据来源

### 主要来源

1. **llama.cpp 性能讨论**
   - URL: https://github.com/ggerganov/llama.cpp/discussions/4167
   - 贡献者: ggerganov (llama.cpp 作者)
   - 内容: 各种设备上的 llama.cpp 性能数据

2. **GPU 基准测试 on LLM Inference**
   - URL: https://github.com/XiongjieDai/GPU-Benchmarks-on-LLM-Inference
   - 贡献者: XiongjieDai
   - 内容: NVIDIA GPU 的 LLM 推理基准测试

### 测试方法

数据通常使用 llama.cpp 收集，这是一个高效的 LLM 推理实现。

**影响测试结果的因素**:
- llama.cpp 版本
- 具体的编译选项
- 操作系统版本
- 系统负载
- 散热情况（持续高负载可能导致降频）

## 如何解读数据

### 选择适合你的设备

根据你的使用场景选择设备：

| 场景 | 推荐关注 | 说明 |
|------|----------|------|
| 实时对话 | Generation 速度 | 需要 30+ tokens/s 流畅体验 |
| 批量处理 | Processing 速度 | 越高越好 |
| 长上下文 | Processing 速度 + 显存 | 两者都重要 |
| 70B 大模型 | 支持的设备 + Generation | 需要高端硬件 |

### 性能预期参考

**实时对话体验**:
- < 10 tokens/s - 较慢，可能卡顿
- 10-30 tokens/s - 可用，但不够流畅
- 30-50 tokens/s - 良好体验
- 50-100 tokens/s - 流畅
- > 100 tokens/s - 非常流畅

**典型设备性能示例**:

| 设备 | 模型 | 量化 | Generation (tokens/s) | 体验 |
|------|------|------|----------------------|------|
| M1 (基础) | Llama3 8B | Q4_K_M | ~10 | 勉强可用 |
| M2 Pro | Llama3 8B | Q4_K_M | ~40 | 良好 |
| M2 Ultra | Llama3 8B | Q4_K_M | ~75 | 流畅 |
| RTX 4090 | Llama3 8B | Q4_K_M | ~125 | 非常流畅 |
| M2 Ultra | Llama3 70B | Q4_K_M | ~12 | 较慢 |
| A100 | Llama3 70B | Q4_K_M | ~22 | 可用 |

### 注意事项

1. **这些是理论最佳值** - 实际使用中可能略低
2. **数据可能过时** - 软件优化持续改进
3. **你的体验可能不同** - 取决于具体配置和使用场景
4. **null 值表示未测试或不支持** - 并不意味着不可能
